工具权限过滤

工具分类 需要最低角色 额外限制
AI 聊天游客无
读取文件游客部门以下只能读本部门/公开文件
写入/修改文件职员部门长以下只能写本部门
删除文件超级管理员仅超级管理员
上传文件职员无
DB SELECT职员部门以下只能查本部门
DB INSERT/UPDATE部门长总经理只读少数表，部门长只写本部门
DB DELETE / DDL超级管理员仅超级管理员
压缩/解压职员部门以下只能操作本部门

请求处理流程

HTTP Request → 提取 cookie 中 token
↓
get_user_by_token(token) → user_id, role, dept_id
↓
get_allowed_tools(role, dept_id) → 过滤后的工具列表
↓
LLM 调用时只发送允许的工具定义
↓
工具执行时 check_permission(tool_name, role, dept_id, path)
↓ (双层验证)
✓ 允许执行
✗ 拒绝执行 + 记录日志

安全设计

  • 双层过滤：LLM 层面隐藏不可用工具 + 执行层面再次校验
  • 文件路径控制：部门用户自动限定在部门子目录
  • DB 表级控制：部分表对低权限角色完全不可见
  • 操作日志：危险操作 (DELETE/DROP) 记录到 audit 表
