数据库表结构设计

在现有 data/app.db 中新增 3 张表

users — 用户表

字段 类型 说明
idINTEGER主键自增
usernameTEXT UNIQUE NOT NULL用户名
emailTEXT UNIQUE邮箱（可选）
password_hashTEXT NOT NULLbcrypt 哈希
roleTEXT NOT NULL DEFAULT 'guest'角色：super_admin/chairman/gm/dept_head/staff/guest
department_idINTEGER所属部门 ID（游客/null 无部门）
is_activeINTEGER DEFAULT 1账号是否启用
created_atREAL注册时间戳

departments — 部门表

字段 类型 说明
idINTEGER主键自增
nameTEXT UNIQUE NOT NULL部门名称
created_atREAL创建时间戳

user_sessions — 登录会话表

字段 类型 说明
tokenTEXT PRIMARY KEYUUID 随机令牌
user_idINTEGER NOT NULL关联用户 ID
created_atREAL登录时间戳
expires_atREAL过期时间戳（默认 24 小时）

设计要点

  • 密码使用 bcrypt 哈希存储
  • user_sessions 与现有 sessions 表（chat 会话）独立
  • session token 通过 httpOnly + Secure Cookie 传递
  • 游客注册时 role='guest', department_id=null
  • 用户-部门 多对一关系
