API 路由设计

在现有 web_server.py 基础上新增的认证和权限相关路由

认证相关 API（公开，无需登录）

方法 路由 说明 请求参数
POST /api/register 注册新用户（默认游客） username, password, email?
POST /api/login 登录，返回 session cookie login, password, remember_me?
POST /api/logout 登出，删除服务端 session (cookie 中的 token)

需要登录的 API（需携带 session cookie）

方法 路由 说明 所需角色
GET /api/me 获取当前用户信息 任意登录用户
POST /api/command AI 对话（已有，加权限检查） 按角色限制工具调用
POST /api/upload 文件上传（已有，加权限检查） 职员及以上
GET/POST /api/admin/* 管理后台 API 超级管理员

页面路由（返回 HTML）

路由 说明 需要登录
/login 登录页面 否
/register 注册页面 否
/chat/ AI 助手主页（现有） 是
/admin/ 管理后台页面 是（超管）

认证流程

1. 用户访问 /chat/ → 检查 cookie 中 token
无 token → 302 跳转到 /login
有 token → 查 user_sessions & users 表验证
有效 → 渲染 chat 页面，注入用户信息
无效/过期 → 清除 cookie，302 跳转 login

2. 每次 /api/command 调用时
根据 user.role 过滤可用的 function calling 工具
只发送权限范围内的 tools 定义给 LLM
